原则：OGNL表达式如果以#开头，访问的contextMap中的数据

如果不以#开头，是访问的根栈中的对象的属性

1.取静态资源
属性：<s:property value= "@java.lang.Integer@MAX_VALUE" />
方法：在struts.xml中配置常量：<constant name="struts.ognl.allowStaticMethodAccess" value="true"></constant>  
             <s:property value= "@java.lang.System@nanoTime()" />
      <s:property value="@com.util.StringUtil@encoder(nickName,'UTF-8')"/>   
2.调用普通对象的普通方法
<!-- JSTL:c:out 功能和 s:property暂时任务输出内容到页面上 .property的value属性的取值就是一个OGNL表达式-->
<s:property value= "'what'.length()"/>
 <s:property value ="'what'.charAt(3)"/>
3.取CompoundRoot中和contextMapz中的数据
OGNL会设定一个根对象（root对象），在Struts2中根对象就是ValueStack值栈） 。如果要访问根对象（即ValueStack）中对象的属性，则可以省略#命名对象，直接访问该对象的属性即可。
Action中代码:
   ValueStack valueStack=ServletActionContext.getContext().getValueStack();

  //set方法放置对象到map中,map再放入到栈(List集合)上
  valueStack.set("student", new Student());
  valueStack.set("employee", new Employee());

  //直接放置对象到栈(List集合)上
  valueStack.getRoot().add(0,new Student());
  valueStack.getRoot().add(1,new Employee());

Jsp页面:
     name:<s:property value="name"/><br>
     age::<s:property value="age"/><br>
     number::<s:property value="number"/><br>
     salary:<s:property value="salary"/><br>

4.ValueStack中取放数据
1）放数据
ValueStack vs = ActionContext.getContext().getValueStack();
·set(String key, Object o)方法
vs.set("p3", "p33");//第一个参数不是ONGL表达式,只是一个普通的字符串.
监测根中栈顶对象是不是一个Map，如果不是，创建一个Map对象的实例，map.put("p","p3"),把Map压入栈顶。
·setValue(String expr, Object value);方法
vs.setValue("p", "p1");//p是一个ONGL表达式.不是以#开头,访问的是root中的对象,从栈顶依次搜索对象的setP()方法，发现没有，报错。
vs.setValue("#p", "p1");//第一个参数是ONGL表达式，以#开头，访问的是contextMap，因此向Map中添加数据。map.put("p","p1");
·vs.push(new Date());
·setParameter(String expr, Object value);

2）取数据
·findString(String expr);
   String s1 = vs.findString("#p"); //获取到的就是一个String，框架已经给你转换好了
·findValue(String expr);
   String s1 = (String)vs.findValue("p");//参数是一个OGNL表达式，如果从栈顶找不到key为p的数据,或者getP()的方法，就从contextMap中找了。不建议，要从map中找加#
   String s2 = (String)vs.findValue("#p");//参数是一个OGNL表达式
Object findValue(String expr):参数是一个OGNL表达式。如果以#开头，从contextMap中找key值所对应的对象。如果不是以#开头，搜索根栈中对象的属性（getter方法）
特别注意：如果编写的表达式不是以#开头，先搜索根栈对象的所有属性，如果没有找到，会把它当做key值到contextMap中找。

3.构造Map,List（重要）
<!--
           规律：
                通用标签：
                     value属性：大部分都是OGNL表达式（90%）
                UI标签：表单有关
                     value属性：大部分都不是OGNL表达式（90%）
                     
                如果要当做OGNL表达式对待：使用%{}
                如果要把OGNL当做字符串对待：使用''
            -->

     </s:iterator >

     <s:iterator value =" #{'a':'aaa','b':'bbb','c':'ccc'}" var= "me">
            <s:property value ="#me.key"/>=< s:property value= "#me.value"/><br />
     </s:iterator >
